Why?
Without recovery, recursive-descent parsers often stop at the first error; panic-mode improves usability by surfacing multiple syntax issues.

Detailed Description

LL(1) recursive-descent parser with panic-mode recovery and AST construction.

The parser consumes a flattened token stream and applies grammar productions as dedicated methods. It records derivation steps and syntax errors while continuing parsing when possible.

Why static state?
Static parser state simplifies mapping from grammar productions to methods and supports existing assignment tooling that expects global parser outputs.
What comes next?
For larger projects, these static members can be migrated into instance state so multiple parser contexts can run concurrently in tests.
