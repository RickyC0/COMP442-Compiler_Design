1/**
2 * @file codegen.h
3 * @brief Moon backend code generation visitor and emission interfaces.
4 *
5 * @details
6 * This header defines the AST-driven backend that lowers typed AST nodes into
7 * Moon assembly. The backend is layout-driven (class/object layouts, function
8 * frames) and uses a register allocator plus stack-based addressing discipline.
9 *
10 * @par Why this shape?
11 * Keeping layout, type-resolution, and emission helpers in one visitor provides
12 * deterministic lowering and consistent diagnostics across all node categories.
13 */
