4/**
5 * @file my_parser.h
6 * @brief Predictive recursive-descent parser interface and parser state contract.
7 *
8 * @details
9 * This header defines the parser used by the compiler pipeline to transform a
10 * token stream into an AST while collecting derivation traces and syntax errors.
11 *
12 * @par Why this shape?
13 * The parser is implemented as static methods plus static state to keep grammar
14 * procedures directly callable as production functions without object plumbing.
15 * This matches course-style LL(1) implementations and makes derivation output
16 * deterministic and easy to inspect.
17 *
18 * @par What comes next?
19 * The AST produced here is consumed by semantic analysis and then code generation.
20 * Any grammar or node-shape changes in this parser should be mirrored in semantic
21 * checks and backend lowering rules.
22 */
