Detailed Description

Immutable lexical token value with scanner utilities.

Each token stores its categorical type, source lexeme, and source line. The same class also hosts scanner routines because token construction and lexical decisions are tightly coupled in this project architecture.

Why keep scanner helpers here?
The design minimizes indirection for students reading generated diagnostics: token construction, classification, and formatting live in one discoverable location.
What comes next?
If lexer complexity grows (for example, DFA tables or unicode handling), these helpers can be moved into a dedicated lexer class while preserving the Token API.
