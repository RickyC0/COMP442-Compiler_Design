Detailed Description

Moon backend code generation visitor and emission interfaces.

This header defines the AST-driven backend that lowers typed AST nodes into Moon assembly. The backend is layout-driven (class/object layouts, function frames) and uses a register allocator plus stack-based addressing discipline.

Why this shape?
Keeping layout, type-resolution, and emission helpers in one visitor provides deterministic lowering and consistent diagnostics across all node categories.
